# Тестовое задание для Node Js разработчика

Нужно создать проект по размещению рекламных роликов на телевидении. 
API должно уметь расставлять ролики в рекламных блоках и выдать в ответ сводную таблицу по расстановке.

В проекте должны быть 4 таблицы в базе данных:
- контракты (contracts). Договоры, заключенный на размещение рекламы 
- рекламные блоки (blocks). Перебивки во время телепрограммы, когда может быть показана реклама
- рекламные ролики (films)
- споты (spots). Кросс-таблица блоков и расставленных роликов

База данных уже в проекте. 2 таблицы готовы и данные в них заполнены, 2 таблицы и модели надо будет добавить.

## Необходимо сделать:

1. Дописать API, в котором будет реализовано два метода GET и POST
2. Метод POST должен принимать следующие параметры в виде JSON: 
```
    {
        "contractID": <ID контракта>,
        "filmName": <Название ролика>,
        "filmDur": <Продолжительность ролика в секундах>
    }
```
После отправки данных должна создаться запись в таблице роликов (Нужно создать модель и таблицу для роликов Film по указанной в JSON структуре). 
После создания записи должен запуститься алгоритм расстановки роликов, который нужно реализовать.

Описание алгоритма:

Данные для расстановки нужно будет взять из таблицы контрактов. contractID передается в POST-запросе. 

Из таблицы contracts по этому ключу взять поле totalRate(общая стоимость контракта) и поле cnlID (ID канала). 

По id канала сделать выборку из таблицы blocks по каналу, это и будут блоки, в которые мы будем размещать рекламу.

На всю доступную стоимость из контракта (contract.totalRate) разместить ролики в блоках канала, указанного в контракте, размещать ролики пока не кончится общая стоимость РК (стоимость конкретного блока в поле rate). 

Размещать два ролика в один блок нельзя. Результат этой расстановки записать в таблицу spots (Создать эту таблицу и модель). Структура blockID, filmID (ссылки на блоки и на ролики)

Требования к алгоритму:
- длительность размещаемого ролика film.filmDur должна быть меньше или равна длительности блока block.blockDur, т.к. он должен туда поместиться (длительность в секундах) 
- Нужно занять блоки с максимально возможным block.rate

!На post запрос выдавать Exception в случаях невозможности выполнения алгоритма по входным данным.!

3. Метод GET должен принимать один параметр ?filmID=<id ролика>, по запросу возвращаются сводные данные в формате:
```
{
    "filmID":<ID ролика>,
    "rate":<Общая набранная стоимость>,
    "dur":<Общая длительность размещенных роликов>
}
```
